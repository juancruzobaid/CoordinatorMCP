Canal de comunicación persistente entre Claude.ai y Claude Code.

- Claude.ai crea instrucciones → Claude Code las ejecuta y reporta


=====================================================================
WORK MODES - SISTEMA DE MODOS DE TRABAJO
=====================================================================

Every work session operates under two dimensions:

1. PROJECT STATE: The active branch (main, dev, feature/x, etc.)
   - Declared once at the start, stays active until changed
   - Affects deployment targets and risk level in all modes
   - Examples: "main" = production, "dev" = safe to experiment

2. WORK MODE: The current activity type (see below)
   - The developer ALWAYS confirms mode changes
   - Claude.ai proposes mode transitions but never switches on its own

---

SESSION START:

At the beginning of every session, the developer declares:
- Branch: "Estamos en [branch]"
- Mode: "Vamos a [evaluar/planificar/ejecutar/revisar]"

Claude.ai confirms:
```
📍 Session active:
- Branch: [branch]
- Mode: [MODE NAME]
- [Restrictions for this mode]

Ready. [Contextual prompt based on mode]
```

---

MODE 1: EVALUAR (Analysis & Decisions)
==========================================

PURPOSE: Analyze alternatives, make architecture decisions, clarify
objectives, evaluate trade-offs, set direction for the project.

✅ ALLOWED:
- Discuss architecture and technical alternatives
- Read code via GitHub API (github_read_file, github_list_files, github_read_tree)
- Analyze technical debt and propose solutions
- Evaluate dependencies, libraries, tools
- Define project direction and priorities
- Review BUSINESS_LOGIC.md to understand current state
- Compare approaches with pros/cons

❌ NOT ALLOWED:
- Create instructions in CoordinatorMCP
- Generate final implementation code
- Deploy or commit anything
- Make changes to production

OUTPUTS: Decisions documented, direction set, clarity achieved.
TYPICAL TRANSITIONS: → PLANIFICAR (when decisions are made)

Claude.ai asks: "¿Pasamos a planificar la implementación?"

---

MODE 2: PLANIFICAR (Implementation Planning)
===============================================

PURPOSE: Define steps, order, batch size, pace. Find the balance
between moving fast and maintaining control of each step.

✅ ALLOWED:
- Read code via GitHub API to understand current state
- Define implementation steps and their order
- Decide batch size (how many changes per cycle)
- Define validation criteria for each step
- Estimate effort and risk per step
- Update MASTER_PLAN.md with the plan
- Define what goes together and what must be separate

❌ NOT ALLOWED:
- Create instructions in CoordinatorMCP (not yet)
- Execute implementations
- Deploy or commit

KEY DECISIONS IN THIS MODE:
- Single implementation or multi-step batch?
- What must work before moving to the next step?
- What's the validation checkpoint for each step?
- What's the rollback plan if something fails?

OUTPUTS: Clear plan with steps, order, and validation criteria.
TYPICAL TRANSITIONS: → EJECUTAR (when plan is confirmed)

Claude.ai asks: "¿El plan está confirmado? ¿Pasamos a ejecutar?"

---

MODE 3: EJECUTAR (Execution & Direction)
===========================================

PURPOSE: Claude.ai becomes the communicator with Claude Code.
Creates instructions, monitors progress, provides direction.

✅ ALLOWED:
- Create instructions in CoordinatorMCP (create_instruction)
- Read code via GitHub API to verify implementations
- Monitor progress (get_project_status, read_progress)
- Provide additional guidance if Claude Code reports issues
- Generate complete code files when needed for instructions
- Read Claude Code's progress reports

❌ NOT ALLOWED:
- Rethink architecture (go back to EVALUAR for that)
- Change the plan substantially (go back to PLANIFICAR)
- Skip validation steps defined in the plan

FLOW:
1. Claude.ai creates instruction(s) per the plan
2. Developer goes to Claude Code and executes
3. Developer returns and says "terminó" or reports issues
4. → REVISAR

Claude.ai tells the developer:
"Instrucción creada [ID]. En Claude Code:
cd ~/mac/[project] → claude → pedile que revise
get_pending_instructions con project_id [id]"

TYPICAL TRANSITIONS: → REVISAR (when Claude Code finishes)
                     → PLANIFICAR (if plan needs adjustment)

---

MODE 4: REVISAR (Review & Close)
====================================

PURPOSE: Verify results, check quality, decide commit/deploy,
close the implementation cycle.

✅ ALLOWED:
- Read progress reports from CoordinatorMCP
- Read modified code via GitHub API (github_read_file)
- Verify that changes match the plan
- Check BUSINESS_LOGIC.md was updated
- Check IMPLEMENTATION_LOG.md has the entry
- Recommend commit, deploy, or fixes
- Flag issues for a new EJECUTAR cycle

❌ NOT ALLOWED:
- Create new instructions (go to EJECUTAR for that)
- Start new features (go to EVALUAR first)
- Deploy without developer confirmation

CHECKLIST:
□ Progress report says "completed"?
□ BUSINESS_LOGIC.md updated?
□ IMPLEMENTATION_LOG.md has entry?
□ Code changes match the plan?
□ Ready to commit?

OUTPUTS: Cycle closed. Decision on next step.
TYPICAL TRANSITIONS:
- → EVALUAR (batch complete, new decisions needed)
- → EJECUTAR (next step in a multi-step plan)
- → PLANIFICAR (need to adjust remaining plan)

Claude.ai asks:
"Revisión completa. ¿Hacemos commit y seguimos con [next step]?"

---

MODE TRANSITIONS DIAGRAM:

    ┌──────────┐     ┌──────────────┐
    │ EVALUAR  │────▶│  PLANIFICAR  │
    │ (1)      │     │  (2)         │
    └──────────┘     └──────┬───────┘
         ▲                  │
         │                  ▼
    ┌──────────┐     ┌──────────────┐
    │ REVISAR  │◀────│  EJECUTAR    │

    └──────┬───┘     └──────────────┘
           │                ▲
           │                │
           └────────────────┘
           (next step in batch)

RULES:
- Developer ALWAYS confirms mode changes
- Claude.ai can SUGGEST a transition but never auto-switch
- Going backwards is allowed (3→2, 4→3) when needed
- Reading code is allowed in ALL modes
- The active branch affects deployment/commit decisions in mode 4

FILE TRANSFER RULES (Claude.ai → Claude Code):

When Claude.ai needs to create or modify files that Claude Code will
save to the repo, follow these rules:

STANDARD (small files, up to ~500 lines, max 3 files):
- Claude.ai puts the file content in the detail_md field of the instruction
- The detail_md MUST start with a manifest header:
  ```
  === FILE MANIFEST ===
  TOTAL FILES: [N]
  1. [path/filename.ext] ([line count] lines)
  2. [path/filename.ext] ([line count] lines)
  === END MANIFEST ===
  ```
- After the manifest, each file is delimited by:
  ```
  === FILE: path/filename.ext ===
  [file content]
  === END FILE: path/filename.ext ===
  ```
- Claude Code MUST verify:
  □ Number of files matches TOTAL FILES in manifest
  □ Each file has both start and end delimiters
  □ No file appears truncated (check last line is the end delimiter)
  □ If any check fails: report as BLOCKED, do not save partial files

EXCEPTION (large files or 4+ files):
- Claude.ai generates downloadable files for the developer
- Claude.ai specifies the exact path in the repo where each file goes
- Developer downloads and copies manually, then tells Claude Code to sync
- This is a fallback, not the default

TYPICAL SESSION FLOW:

1. EVALUAR: "Quiero agregar [feature X]"
   - Claude.ai analyzes, discusses alternatives, reads code if needed
   - Decision made on approach
   - Developer: "OK, pasemos a planificar"

2. PLANIFICAR: Claude.ai proposes implementation steps
   - Developer reviews, adjusts batch size and order
   - Plan confirmed with validation checkpoints
   - Developer: "Dale, pasemos a ejecutar"

3. EJECUTAR: Claude.ai creates instructions in CoordinatorMCP
   - Developer goes to Claude Code (cd to project → claude → paste instruction)
   - Claude Code executes and reports progress
   - Developer returns: "Terminó"

4. REVISAR: Claude.ai checks CoordinatorMCP progress
   - Verifies code, docs, and quality
   - Developer confirms commit/deploy
   - Developer: "Listo, siguiente paso" → back to EJECUTAR (next step)
   - Or: "Terminamos el batch" → back to EVALUAR

QUICK MODE SWITCHES:
- "Pasemos a evaluar" → EVALUAR
- "Pasemos a planificar" → PLANIFICAR  
- "Pasemos a ejecutar" → EJECUTAR
- "Terminó, revisemos" → REVISAR
- "Cambiamos a branch [name]" → Updates project state


=====================================================================
SUMMARY OF RESPONSIBILITIES
=====================================================================

CLAUDE.AI:
- Strategic planning
- Generation of complete code for small/medium tasks
- Creation of instructions via CoordinatorMCP for large tasks
- Monitoring progress via CoordinatorMCP
- Reading project code directly from GitHub via github_read_file, github_list_files, github_read_tree
  (no need to ask the developer to paste code)
- Implementation Audits
- Maintenance of PROJECT_CONTEXT.md and MASTER_PLAN.md
- Review of BUSINESS_LOGIC.md for accuracy
- Technical support for developers

CLAUDE CODE:
- Check CoordinatorMCP for pending instructions on startup
- Implementation of tasks according to instructions
- Automatic updating of IMPLEMENTATION_LOG.md
- Automatic updating of BUSINESS_LOGIC.md (functional changes only)
- Commit and push after each completed task
- Execution of installation/configuration commands
- Modification of multiple files simultaneously
- Report progress/completion via CoordinatorMCP submit_progress
- Does NOT make architectural decisions, only executes

DEVELOPER:
- Defines which features to implement
- Decides when to use Claude.ai vs. Claude Code
- Starts Claude Code in the correct project directory first,
  then pastes instructions inside (NEVER inline with claude "...")
- Reviews and approves changes
- Makes commits and pushes to GitHub
- Requests documentation updates when necessary
- Reviews BUSINESS_LOGIC.md periodically to verify accuracy

COORDINATORMCP:
- Persistent communication channel between Claude.ai and Claude Code
- Stores instructions, progress reports, and file references
- Isolates data per project via project_id
- Available at: https://mcp.juancruz.com.ar/mcp

FINAL NOTES

- The 4 MD files are the ONLY source of shared truth.
- CoordinatorMCP is the ONLY channel for task assignment and progress.
- Everything must be in the MD files to maintain synchronization.
- Each project MUST have a unique project_id in CoordinatorMCP.
- Claude Code MUST be started first (cd to project, then run claude),
  NEVER with inline instructions (claude "instruction").
- Review IMPLEMENTATION_LOG.md weekly to detect patterns.
- Review BUSINESS_LOGIC.md every 10-15 features for accuracy.
- Update documentation proactively, not reactively.
- This system is language/framework agnostic.

INSTRUCTIONS FOR USING THIS FILE
